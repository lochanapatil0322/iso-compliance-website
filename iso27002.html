ISO/IEC 27002: Understanding Security Controls

ISO/IEC 27002 provides guidance for information security controls. It supports ISO 27001 by helping teams understand how controls can be selected, designed, and operated.

How ISO 27001 and ISO 27002 Differ

Standard Purpose Practical Use
ISO 27001 Defines requirements for an ISMS. Used for certification, audit planning, governance, risk management, and continual improvement.
ISO 27002 Provides guidance for information security controls. Used by security teams to understand control intent, implementation ideas, and operational practices.

Main Control Categories

Organizational

Policies, responsibilities, supplier security, threat intelligence, project security, and incident management.

People

Awareness training, screening, confidentiality, remote work, disciplinary processes, and role changes.

Physical

Secure areas, equipment protection, clear desk practices, facility access, cabling, and environmental safeguards.

Technological

Access control, malware protection, logging, backup, encryption, network security, and secure development.

Example Controls

  • Password and authentication rules should match the sensitivity of systems and user roles.
  • Backups should be scheduled, protected, monitored, and tested for restoration.
  • Logging and monitoring should help detect suspicious activity and support investigation.
  • Supplier access should be controlled through agreements, reviews, and defined responsibilities.
  • Security awareness should be repeated so employees understand current risks and expected behavior.
